Algorithms and Data Structures Narrative

The artifact used for all of my enhancements is the full-stack website built during my full-stack development course. This enhancement focused on adding a swiss style bracket system for running TCG tournaments through the website. This required adding several new features to the existing base, and refactoring some of the existing ones. The enhancement showcases my ability with algorithms and data structures because I am utilizing a swiss pairing algorithm that pairs players based on performance, avoids repeat pairings, and must assign byes in the case of odd numbers of players. It also calculates tie-breakers using metrics like the win% of opponents played.

I feel I have met the outcomes I planned to meet with this enhancement. I had to use an algorithm to solve the swiss pairing system, and I created a real user-flow that implements user roles (organizers can create and alter events, users can register for those events). I also did a pass at the front-end, attempting to get closer to a final product that demonstrates user flow (as opposed to everything existing in API layers… which a lot of it still does, so there’s more work to be done on that front).

This whole task has been very challenging. As I mentioned previously, the front-end still needs to catch up in functionality. There’s still a lot of work to be done to make this a functional live product, as opposed to its current state. The main lesson of this enhancement has been that tasks that seem simple can often be very complicated. From implementing all of the components needed for the swiss event system, to wiring it cleanly to the event system, to making front-end changes necessary for organizers to create, edit, and advance their events. Every time I would think I was finished, there’d always be another thing to do, or another thing to hook up. In the future, when planning out tasks, I need to take more time with the planning phase and get into the minutia of each task.
